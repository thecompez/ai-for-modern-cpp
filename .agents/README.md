# Repository Agent Workflows

The skills in this directory are adapters around the canonical contract in
`AGENTS.md`. They do not redefine repository policy.

| Skill | Purpose |
|---|---|
| `source-command-implement` | Implement the smallest verified change |
| `source-command-add-module` | Add a documented module boundary |
| `source-command-design-qt-quick-ui` | Select and implement a Qt Quick primary UI with a clean C++ boundary |
| `source-command-test` | Configure, build, and test honestly |
| `source-command-review` | Produce rule-driven actionable review |
| `source-command-reflect` | Convert durable corrections into knowledge |
| `source-command-release` | Verify and prepare an approved release |

Every skill must route through `AGENTS.md` and the task-specific guides under
`docs/agent/`.
