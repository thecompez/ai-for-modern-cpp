# Review

Read `AGENTS.md` and `docs/REVIEW.md`, inspect the current diff, then read every
task guide touched by the change. Prioritize correctness, safety, ownership,
architecture, and real verification. Cite stable rule identifiers for each
actionable finding and keep line ranges tight. Do not edit unless asked.
